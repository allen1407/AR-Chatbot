# 🐚 Shankh AR — Multilingual AR Financial Assistant

**A RAG-powered Augmented Reality financial chatbot that speaks 22 Indian languages.**  
Point your camera at a marker → a 3D avatar appears → ask about stocks in any Indian language → get live market data spoken back in your language.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🎯 **AR Marker Tracking** | MindAR image tracking — stable, no drift |
| 🧍 **3D Animated Avatar** | GLTF avatar with idle/think/talk animations |
| 💬 **Lip Sync** | Rhubarb-based mouth animation synced to speech |
| 📊 **Live Stock Data** | Real-time NSE/BSE prices + analyst signals |
| 🤖 **RAG Pipeline** | Retrieval-Augmented Generation from your knowledge base |
| 🌐 **22 Indian Languages** | Auto-detects language from speech or script |
| 🔊 **Bhashini TTS** | Government of India's Bhashini API for native speech |
| 📱 **Mobile-First** | Works on iOS Safari and Android Chrome |

---

## 🏗️ Architecture

```
User speaks (any Indian language)
        ↓
SpeechRecognition API (browser)
        ↓
GPT-4o-mini detects language (ISO code)
        ↓
RAG retrieval + Live stock API (parallel)
        ↓
GPT-4o-mini answers in detected language
        ↓
Bhashini TTS (Dhruva API) → WAV audio
        ↓
Rhubarb lip-sync → 3D avatar mouth animation
        ↓
AR avatar speaks in your language 🎙️
```

### Tech Stack

| Layer | Technology |
|---|---|
| AR Engine | [MindAR](https://github.com/hiukim/mind-ar-js) 1.2.5 |
| 3D Engine | [A-Frame](https://aframe.io/) 1.5.0 + Three.js r158 |
| Avatar | GLTF/GLB with DRACO compression |
| LLM | OpenAI GPT-4o-mini |
| RAG | Custom vector retrieval server |
| TTS | [Bhashini Dhruva API](https://bhashini.gov.in/) |
| Lip Sync | Rhubarb (server-side) |
| Stock Data | Custom NSE/BSE API backend |

## 🔑 API Keys

You need **3 API keys** to run this project:

### 1. OpenAI API Key
- Go to [platform.openai.com](https://platform.openai.com/api-keys)
- Create a new secret key
- Used for: GPT-4o-mini (answers), language detection

### 2. Bhashini User ID + API Key
- Register at [bhashini.gov.in/ulca](https://bhashini.gov.in/ulca/user/register)
- After login → go to **My Profile** → **API Keys**
- Copy your **User ID** and **ULCA API Key**
- Used for: Text-to-Speech in 22 Indian languages via Dhruva

### 3. Stock API (optional)
- Your own NSE/BSE data provider or use the mock server

## 🚀 Setup

### Prerequisites

- Python 3.10+
- Node.js 18+ (for DRACO/GLB tools, optional)
- [ngrok](https://ngrok.com/) (to expose local server to mobile)
- A device with a camera (mobile recommended)

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/shankh-ar.git
cd shankh-ar
```

### 2. Set up the backend server

```bash
cd server
pip install -r requirements.txt
```

Create your `.env` file:

```bash
cp .env.example .env
```

Edit `.env`:

```env
OPENAI_API_KEY=sk-...your-key-here...
BHASHINI_USER_ID=your-ulca-user-id
BHASHINI_API_KEY=your-ulca-api-key
```

### 3. Start the server

```bash
python combined_server.py
# Server starts on http://localhost:8000
```

### 4. Expose with ngrok (for mobile testing)

```bash
ngrok http 8000
# Copy the https://xxxx.ngrok-free.app URL
```

### 5. Configure the frontend

Open `index.html` and update the `CONFIG` block:

```javascript
const CONFIG = {
  OPENAI_API_KEY   : 'sk-...your-key...',   // ← replace
  BHASHINI_USER_ID : 'your-user-id',         // ← replace
  BHASHINI_API_KEY : 'your-api-key',         // ← replace
};
```

> **Note:** `SERVER_URL` is automatically set to `window.location.origin` — no change needed if serving `ar_new.html` from the same server.

### 6. Serve the frontend

Serve `ar_new.html` from your Python server (it should already be configured), then open the ngrok URL on your mobile device.

### 7. Print the marker

Print `marker.jpg` or display it on a second screen. Point your mobile camera at it to summon Shankh.

## 🔊 Bhashini TTS Flow

```
1. getBhashiniServiceId(langKey)
   └─ POST /ulca/apis/v0/model/getModelsPipeline
      Headers: userID, ulcaApiKey
      → returns: serviceId, callbackUrl, inferenceKey

2. bhashiniTTS(text, langKey)
   └─ POST {callbackUrl}/services/inference/pipeline
      Headers: Authorization: {inferenceKey}
      Body: { pipelineTasks: [{ taskType: 'tts', config: { serviceId, language, gender } }] }
      → returns: base64 WAV audio

3. Decode base64 → Blob → Audio element → play()
```

**Fallback chain:** Bhashini TTS → Rhubarb/OpenAI TTS → Browser SpeechSynthesis

## 🔒 Security Notes

> ⚠️ **Never commit real API keys to GitHub.**

Before pushing:
1. Replace all API keys in `ar_new.html` with placeholder strings
2. Use environment variables for server-side keys
3. Add `.env` to `.gitignore`

The `index.html` in this repo has placeholders:
```javascript
OPENAI_API_KEY : 'YOUR_OPENAI_API_KEY_HERE',
BHASHINI_USER_ID : 'YOUR_BHASHINI_USER_ID_HERE',
BHASHINI_API_KEY : 'YOUR_BHASHINI_API_KEY_HERE',
```
